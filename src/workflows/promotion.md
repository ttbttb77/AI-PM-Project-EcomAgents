# 促销活动策划执行工作流

## 流程概述

商家提出活动需求 → 运营分析数据制定方案 → 美工/文案准备素材 → 方案审核 → 多平台上线 → 效果追踪

**传统方式**：1 周（手动拉数据 + 写方案 + 找设计出图 + 逐平台配置）
**Agent 协作**：~1 天 | **效率提升 85%**

---

## 触发条件

```
"做个 618 活动"
"促销活动：全店满 200 减 30"
"帮我策划一个双 11 方案"
"给这款商品搞个限时秒杀"
```

---

## 执行流程

### 第 1 步：需求确认（店长 Agent | ~5 分钟）

| 确认项 | 示例 |
|--------|------|
| 活动目标 | 清库存 / 冲 GMV / 拉新 / 品牌曝光 |
| 活动预算 | ¥5,000 |
| 时间范围 | 6 月 1 日 - 6 月 18 日 |
| 参与商品 | 全店 / 指定 20 款 |
| 活动平台 | 淘宝 + 抖音 |
| 预期 ROI | ≥ 3.0 |

### 第 2 步：数据分析（运营 Agent | ~30 分钟）

```
分析维度：
├─ 历史活动数据（去年 618、今年日常促销）
├─ 竞品活动策略（满减/折扣/赠品组合）
├─ 商品销售趋势（哪些商品适合做活动）
├─ 用户画像（目标用户的购买偏好）
└─ 预算测算（不同折扣力度的 GMV 和利润预估）
```

### 第 3 步：方案制定（运营 Agent | ~20 分钟）

输出活动方案：
- 活动主题和玩法（满减 / 折扣 / 赠品 / 秒杀）
- 参与商品清单和折扣力度
- 分阶段执行节奏（预热期 → 爆发期 → 返场期）
- 推广渠道和预算分配
- 预期 GMV / ROI / 新增会员数

### 第 4 步：方案审核（店长 Agent | ~10 分钟）

店长审核要点：
- ROI 预估是否合理
- 预算是否在商家可接受范围
- 是否有超卖/亏损风险
- 与商家确认关键决策点

### 第 5 步：素材准备（美工 + 文案并行 | ~4 小时）

**🎨 美工**：
- 活动主海报 ×1
- 各渠道 Banner（淘宝/京东/抖音尺寸）
- 商品图活动角标
- 详情页活动装饰

**✍️ 文案**：
- 活动主题文案
- 推广话术（短信/推送/直播间）
- 优惠券引导文案
- 社交媒体推广文案

### 第 6 步：活动上线（运营 Agent + MCP | ~1 小时）

```
MCP 工具 → 配置各平台活动规则
├─ 淘宝：满减规则 + 优惠券 + 店铺装修
├─ 京东：促销标签 + 满减 + 优惠券
└─ 抖音：限时秒杀 + 优惠券 + 直播间配置
```

### 第 7 步：效果追踪（运营 Agent | 持续）

| 监控指标 | 频率 | 告警 |
|---------|:---:|------|
| 实时 GMV | 每 5 分钟 | 低于预期 30%+ |
| 转化率 | 每小时 | 异常下降 |
| 预算消耗 | 每小时 | 超过预算 80% |
| ROI | 每日 | 低于目标 |
| 库存 | 实时 | 超卖风险 |

## 促销模板库

| 类型 | 模板 | 适用场景 |
|------|------|---------|
| 满减 | 满 X 元减 Y 元 | 提升客单价 |
| 折扣 | 全场/指定 Z 折 | 清库存 |
| 赠品 | 买 A 送 B | 新品推广 |
| 秒杀 | X 小时限时价 | 引流拉新 |
| 优惠券 | 满 X 减 / 无门槛 | 日常促销 |
| 多件多折 | 2 件 8 折 / 3 件 7 折 | 提升连带率 |

---

## 异常处理

| 异常 | 处理 |
|------|------|
| 活动效果远低于预期 | 运营 Agent 分析原因 → 建议调整策略 → 商家确认后执行 |
| 预算超支 | 立即告警 → 暂停高消耗渠道 → 商家决策是否追加 |
| 库存不足 | 仓储 Agent 联动 → 暂停超卖商品推广 → 建议补货 |
| 薅羊毛风险 | 规则漏洞检测 → 自动修复 → 异常订单标记 |
