# 新品上架自动化工流

## 流程概述

商家提供商品信息 → 店长分解任务 → 美工/文案/运营/仓储并行协作 → 质审 → 一键上架 → 汇报

**传统方式**：2 天（运营写文案 + 找设计出图 + 手动上架 + 设置库存）
**Agent 协作**：~25 分钟 | **效率提升 95%**

---

## 触发条件

商家通过店长 Agent 发送指令：

```
"我要上架一个 XXX"
"上架新商品：XXX"
"新品：XXX，面向 XXX 人群，价格 XXX 左右"
"帮我上架这个：XXX"
```

---

## 执行流程

### 第 1 步：信息收集（店长 Agent | ~2 分钟）

店长引导商家确认关键信息：

| 必填信息 | 示例 | 选填信息 |
|---------|------|---------|
| 商品名称 | 304 不锈钢保温杯 | 品牌 |
| 商品类目 | 家居 > 杯具 > 保温杯 | 型号 |
| 核心卖点 | 12h 保温、便携防漏 | 材质 |
| 目标人群 | 商务白领 25-40 岁 | 风格要求 |
| 价格范围 | ¥79-99 | 参考竞品链接 |
| 初始库存 | 500 件 | 安全库存阈值 |
| 上架平台 | 淘宝 | 多平台同步 |

> **设计原则**：店长只问一次，不要反复追问。信息不全时用默认值填充并在汇报时标注。

### 第 2 步：任务分解与分配（店长 Agent | ~1 分钟）

```
📋 任务分解
┌──────────────────────────────────────┐
│ Phase 1 [并行]                        │
│ ├─ 🎨 美工 → 生成 5 张商品图          │
│ ├─ ✍️ 文案 → 撰写标题+详情页+卖点      │
│ └─ 📊 运营 → 竞品分析+定价策略         │
│                                       │
│ Phase 2 [依赖 Phase 1 运营完成]        │
│ └─ 📦 仓储 → 设置 SKU+初始库存+安全阈值 │
│                                       │
│ Phase 3 [依赖全部完成]                 │
│ └─ 📊 运营 → MCP 推送到目标平台上架     │
└──────────────────────────────────────┘
```

### 第 3 步：并行处理（~15 分钟）

#### 🎨 美工（~5 分钟）
```
输入：商品名称 + 类目 + 风格要求 + 目标人群 + 平台
过程：生成 15-20 张候选 → 筛选 5 张最优 → 精修优化
输出：
  - 01_主图.jpg（白底正面）
  - 02_细节图.jpg（材质特写）
  - 03_场景图.jpg（办公桌场景）
  - 04_对比图.jpg（与普通保温杯对比）
  - 05_包装图.jpg（礼盒展示）
```

#### ✍️ 文案（~3 分钟）
```
输入：商品名称 + 卖点 + 目标人群 + 平台
过程：生成标题 ×5 → 详情页文案 800+ 字 → 禁用词过滤
输出：
  - 标题版本 1-5（不同风格）
  - 详情页文案（痛点→卖点→参数→场景→背书→CTA）
  - SEO 关键词列表
```

#### 📊 运营（~8 分钟）
```
输入：商品类目 + 成本 + 目标人群 + 平台
过程：分析竞品价格 → 计算定价 → 设置促销
输出：
  - 竞品价格分析表
  - 推荐定价 ¥89（利润率 45%）
  - 上架促销策略
```

### 第 4 步：库存设置（仓储 Agent | ~2 分钟）

```
输入：商品信息 + 初始库存数
过程：创建 SKU → 设置初始库存 → 配置安全阈值
输出：
  - SKU 编号
  - 库存 500 件
  - 安全库存阈值 50 件（低于此值告警）
```

### 第 5 步：质量审核（店长 Agent | ~2 分钟）

| 审核项 | 标准 |
|--------|------|
| 商品图 | 数量 ≥ 5 张，尺寸合规，无模糊 |
| 文案 | 无禁用词，字数合规，卖点准确 |
| 定价 | 有数据依据，利润率合理 |
| 库存 | SKU 正确，库存数正确，有安全阈值 |

### 第 6 步：一键上架（运营 Agent + MCP | ~3 分钟）

```
调用 MCP 工具 publish_product → 推送到目标平台
  ├─ 成功 → 记录上架时间 + 平台商品 ID
  └─ 失败 → 自动重试（最多 3 次）→ 仍失败则告警 + 人工介入
```

### 第 7 步：结果汇报（店长 Agent | ~1 分钟）

```
✅ 新品上架完成！
━━━━━━━━━━━━━━━━━━━━━━━━━━
📦 304 不锈钢保温杯 — 已上架
├─ 平台：淘宝
├─ 链接：https://item.taobao.com/...
├─ 商品图：5 张（800×800）
├─ 文案：标题 ×5 版本 + 详情页 1,200 字
├─ 定价：¥89（利润率 45%）
├─ 库存：500 件（安全阈值 50 件）
├─ 耗时：22 分钟
└─ 状态：✅ 已上线

💡 建议：竞品均价 ¥99，建议 1 周后根据数据调整至 ¥95
```

---

## 异常处理

| 异常 | 处理 |
|------|------|
| 美工出图质量不达标 | 退回重出，附具体调整要求，最多 2 次 |
| 平台 API 上架失败 | 自动重试 3 次，仍失败则记录原因 + 告警 |
| 商家信息不全 | 用默认值填充，在汇报中标注"需商家确认"的项目 |
| 模型 API 不可用 | 自动 Failover 到备用模型 |

## 配置参数

```yaml
new-product:
  agents: [manager, designer, copywriter, operator, warehouse]
  parallel: true
  timeout: 1800          # 30 分钟
  max_retries: 3
  retry_delay: 5000      # 5 秒
```
