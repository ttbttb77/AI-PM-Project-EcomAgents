# 日常运营自动化工作流

## 流程概述

定时/事件驱动的自动化运营：客服自动回复 → 财务记账 → 库存巡检 → 数据分析 → 日报推送

**传统方式**：商家每天手动检查，耗时 4 小时+
**Agent 协作**：全自动 | **效率提升 90%+**

---

## 触发方式

| 服务 | 触发方式 | 频率 |
|------|---------|------|
| 客服自动回复 | 事件驱动（收到新消息） | 实时 |
| 财务记账 | 事件驱动（新订单产生） | 实时 |
| 库存巡检 | 定时 (cron: 0 */2 * * *) | 每 2 小时 |
| 运营数据分析 | 定时 (cron: 0 9 * * *) | 每天早上 9:00 |
| 日报生成推送 | 定时 (cron: 0 22 * * *) | 每天晚上 22:00 |

---

## 执行详情

### 1. 客服自动回复（Service Agent | 实时）

```
客户消息 → 意图识别 → RAG 检索 → 生成回复
    │
    ├─ 高置信度（≥70%）→ 自动回复
    ├─ 低置信度（<70%）→ 创建工单 + 人工客服
    └─ 愤怒情绪 → 立即转人工
```

### 2. 财务记账（Finance Agent | 事件驱动）

```
新订单 → 自动记录：
├─ 订单编号 + 金额 + 平台 + 时间
├─ 更新今日收入/订单量/客单价
└─ 退款发生时 → 自动冲销
```

### 3. 库存巡检（Warehouse Agent | 每 2 小时）

```
遍历所有 SKU：
├─ 库存 < 安全阈值 → 补货提醒 + 建议补货量
├─ 库存 < 0 → 紧急告警 + 自动暂停推广
├─ 90 天无动销 → 滞销预警
└─ 库存健康度评分
```

### 4. 运营数据分析（Operator Agent | 每日 9:00）

```
多平台数据聚合：
├─ 昨日核心指标：GMV / 订单量 / 客单价 / 转化率
├─ 7 日趋势对比
├─ 商品排行（TOP 10 / BOTTOM 10）
├─ 费用汇总（推广 + 物流 + 其他）
└─ 异常检测：指标波动 > 20% → 归因分析
```

### 5. 日报生成推送（Manager Agent | 每日 22:00）

```markdown

━━━━━━━━━━━━━━━━━━━━━━━━━━

📦 订单：142 单（↑8%）
👤 客单价：¥88.6（↑4%）
📈 转化率：3.8%（↑0.2%）

🏆 TOP 3 商品：
1. 304 保温杯 — ¥3,560（28 单）

3. 无线充电器 — ¥1,960（35 单）

• 客服自动回复 86 条，自动解决率 68%
```

## 异常告警

| 告警 | 触发条件 | 通知方式 |
|------|---------|---------|
| 超卖风险 | 库存 < 0 | 🔴 电话 + 飞书 |
| 客服满意度骤降 | 👍 率 < 80% | 🟡 飞书 |
| 财务异常 | 单笔退款 > ¥1,000 | 🟡 飞书 |
| 库存不足 | SKU < 安全阈值 | 🟢 系统通知 |
| 销售骤降 | 昨日 GMV < 7 日均值 × 0.5 | 🟡 飞书 |

---

## 配置参数

```yaml
daily-ops:
  agents: [service, finance, warehouse, operator, manager]
  triggers:
    - type: event
      service: service      # 客服：事件驱动
    - type: event
      service: finance      # 财务：事件驱动
    - type: cron
      cron: "0 */2 * * *"   # 库存：每 2 小时
      service: warehouse
    - type: cron
      cron: "0 9 * * *"     # 运营分析：每日 9:00
      service: operator
    - type: cron
      cron: "0 22 * * *"    # 日报：每日 22:00
      service: manager
```
