## 一、RICE 评分模型说明

### 1.1 评分维度

| 维度 | 英文 | 说明 | 评分标准 |
|-----|------|------|---------|
| **覆盖范围** | Reach | 多少用户/角色会受益 | 1-10 分，10=覆盖最广 |
| **影响力** | Impact | 对业务目标的影响程度 | 1-10 分，10=影响最大 |
| **信心度** | Confidence | 对估算的信心程度 | 50%-100%，100%=完全确定 |
| **投入成本** | Effort | 实现所需工作量 | 人周数，数值越大成本越高 |

### 1.2 计算公式

```
RICE Score = (Reach × Impact × Confidence) / Effort
```

**分数解读**：
- 分数越高，优先级越高
- 分数<3：建议延后或重新评估
- 分数 3-6：中等优先级
- 分数>6：高优先级，建议优先投入

## 七、风险与缓解

### 7.1 风险评估

| 风险 | 概率 | 影响 | 缓解措施 |
|-----|------|------|---------|
| 技术迭代过快 | 高 | 中 | 采用模块化架构，便于替换 |
| 平台 API 变更 | 中 | 高 | 屏幕语义理解技术降低 API 依赖 |
| 安全合规风险 | 中 | 高 | 早期引入安全设计，通过等保认证 |
| 客户接受度低 | 中 | 中 | 从高 ROI 场景切入，快速证明价值 |
| 数据质量不足 | 高 | 中 | 建立数据清洗与校验机制 |
