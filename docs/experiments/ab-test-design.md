## 实验 1：客服置信度阈值优化

### 背景
当前客服 Agent 置信度阈值设为 **70%**（<70% 转人工）。此阈值来自行业经验值，未经过数据验证。

### 假设
将阈值从 70% 调整为 **60%**，可以在**不显著增加错误率**的前提下，**提升自动解决率 8-10%**，减少不必要的人工转接。

### 实验设计

| 维度 | A 组（对照组） | B 组（实验组） |
|------|:---------:|:---------:|
| 置信度阈值 | 70%（当前） | 60% |
| 用户占比 | 50% | 50% |
| 实验周期 | 2 周 | 2 周 |

### 核心指标

| 指标 | 优先级 | 预期 | 统计方法 |
|------|:-----:|------|---------|
| **自动解决率** | 主指标 | B 组 ≥ A 组 + 8% | 双样本 Z 检验 |
| **错误回复率** | 护栏指标 | B 组 ≤ A 组 + 2% | 双样本 Z 检验 |
| **客户满意度** | 辅指标 | B 组 ≥ A 组 - 3% | t 检验 |
| **转人工率** | 观察指标 | B 组 ≤ A 组 - 10% | 描述性统计 |

### 决策规则
- ✅ B 组自动解决率显著提升 **且** 错误率未显著恶化 → 全量切换
- ⚠️ B 组自动解决率提升但错误率也显著增加 → 尝试中间值 65%
- ❌ B 组无显著差异 → 维持 70%

## 实验 2：新品上架"风格模板先行" vs "自由发挥"

### 背景
当前新品上架工作流中，美工 Agent 直接根据商品信息自由创作。但 PRD §4.2.8 提到可选预设风格模板。

### 假设
在上架流程 Step 1 增加**风格模板选择**（简约/商务/可爱/科技），可以让美工 Agent 的**首次通过率提升 15%**，减少质审退回次数。

### 实验设计

| 维度 | A 组（对照组） | B 组（实验组） |
|------|:---------:|:---------:|
| 流程 | Step 1 直接填信息 → Step 3 美工自由发挥 | Step 1 增加风格模板选择 → 美工按模板创作 |
| 用户占比 | 60% | 40%（新功能谨慎放量） |
| 实验周期 | 1 周 | 1 周 |

### 核心指标

| 指标 | 优先级 | 预期 |
|------|:-----:|------|
| **首次输出通过率**（质审一次通过） | 主指标 | B 组 ≥ A 组 + 15% |
| **总完成时长** | 辅指标 | B 组 ≤ A 组（不因新增步骤而变慢） |
| **商家满意度** | 观察指标 | B 组 ≥ A 组 |
| **风格模板使用率** | 过程指标 | ≥ 80% |

## 实验 3：并行 Agent 数量对效率的影响

### 背景
当前新品上架「并行处理」步骤同时启动 3 个 Agent（美工+文案+运营）。理论上增加并行度可以进一步缩短总耗时。

### 假设
将库存设置（Warehouse）也纳入并行步骤，从 **3 Agent 并行 → 4 Agent 并行**，总耗时缩短 **~15%**。

### 实验设计

| 维度 | A 组（3 Agent 并行） | B 组（4 Agent 并行） |
|------|:---------:|:---------:|
| 并行 Agent | 美工+文案+运营 | 美工+文案+运营+仓储 |
| 实验周期 | 1 周 | 1 周 |

### 核心指标

| 指标 | 优先级 | 预期 |
|------|:-----:|------|
| **并行步骤耗时** | 主指标 | B 组 ≤ A 组 - 15% |
| **总工作流耗时** | 主指标 | B 组 ≤ A 组 - 10% |
| **API 并发错误率** | 护栏指标 | B 组 ≤ A 组 + 1% |
| **Token 消耗峰值** | 观察指标 | 监控是否超预算 |

## 实验管理规范

| 规范项 | 要求 |
|--------|------|
| **实验文档** | 存于 `docs/experiments/`，命名格式 `exp-{序号}-{描述}.md` |
| **评审机制** | 每个实验上线前需产品+算法+研发三方评审 |
| **停止规则** | 护栏指标恶化 → 立即停止；主指标连续 3 天无趋势 → 提前终止 |
| **知识沉淀** | 实验结果（无论成败）写入对应 Agent 的 SOUL.md Pitfalls 区域 |
