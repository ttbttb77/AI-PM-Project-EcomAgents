P0 EXP-01：客服置信度阈值优化

假设：如果我们将客服 Agent 置信度阈值从 70% 调整为 60%，那么自动解决率会从 60% 提升至 68-70%，因为 60-70% 置信度区间的回复在人工审核中正确率达 92%，当前被不必要地转人工了。
MDE：自动解决率 60% → 68%（相对 +13.3%，绝对 +8pp）

实验设计

分组方式用户级分流（按 customer_id hash）
流量分配A组(对照组) 50% / B组(实验组) 50%
实验周期2 周（14天，含 2 个完整周）
预热期2 天（数据稳定后开始计算）

P1 EXP-02：新品上架"风格模板先行"

假设：如果在信息采集阶段增加风格模板选择（简约/商务/可爱/科技），那么美工 Agent 的首次通过率会从 78% 提升至 90%+，因为 风格模板消除了"美工自由发挥方向偏差"这一最大的质审退回原因。
MDE：首次通过率 78% → 90%（相对 +15.4%，绝对 +12pp）

分组方式用户级分流（按 user_id hash）
流量分配A组(自由发挥) 40% / B组(模板先行) 40% / 预留 20%（缓冲新用户）
实验周期1 周（覆盖工作+周末上架场景）
混杂控制按商品类目分层（服装/数码/家居），确保两组类目分布一致

每组 72 · 共计 144 次上架

P1 EXP-03：3→4 Agent 并行扩展

假设：如果新品上架的"并行处理"步骤从 3 Agent（美工+文案+运营）扩展到 4 Agent（+仓储），那么并行步骤耗时会从 ~8s 降至 ~7s，因为 仓储设置库存不依赖其他 Agent 的输出，可以安全并行化。
MDE：并行步骤耗时 8.0s → 6.8s（相对 -15%，绝对 -1.2s）

❓ FAQ

Q：Demo 阶段日活很低，样本量不够怎么办？
A：上表中的样本量是基于统计公式的理论值。Demo 阶段优先做 EXP-02/03（样本量需求小，20-70 次即可），EXP-01 等有真实用户流量后再做。当前可用定性评估替代：人工审核 60-70% 置信度区间的回复质量。

Q：如果多个实验同时跑，怎么避免互相干扰？
A：EXP-02（模板先行）和 EXP-03（4并行）操作的是同一个工作流的不同环节，需互斥分流——一个用户同时只能在一个实验中。建议先跑 EXP-02，结束后再跑 EXP-03。
