# 电商多 Agent 系统 — 指标体系定义

> 基于 PRD v1.1 §11 评测体系 + §16 验收标准，融合 tutorial Phase 4 三层指标体系

---

## 指标体系总览

```
┌──────────────────────────────────────────────────────────┐
│ 第一层：业务结果指标（老板/商家关心的）                      │
│ ├─ 工作流完成率 / 平均耗时 / 人效提升比 / 商家活跃度         │
├──────────────────────────────────────────────────────────┤
│ 第二层：过程质量指标（PM 关注的）                           │
│ ├─ 意图识别准确率 / 任务分发准确率 / 首次输出通过率           │
├──────────────────────────────────────────────────────────┤
│ 第三层：技术健康指标（研发关注，PM 也要了解）                 │
│ ├─ API调用成功率 / Token消耗 / Agent响应延迟 / 幻觉率       │
└──────────────────────────────────────────────────────────┘
```

---

## 第一层：业务结果指标

| 指标名 | 定义 | 计算公式 | 目标值 | 数据来源 | 更新频率 |
|--------|------|---------|:-----:|---------|:------:|
| **工作流完成率** | 成功完成的工作流占总发起数的比例 | 完成数 ÷ 发起数 × 100% | ≥ 95% | workflow_executions 表 | 每日 |
| **平均完成时长** | 从发起到完成的平均耗时 | Σ(完成时间-发起时间) ÷ 完成数 | ≤ 30min（新品上架） | workflow_executions 表 | 每日 |
| **人效提升比** | 传统人工耗时 vs Agent 耗时 | 传统耗时 ÷ Agent 耗时 | ≥ 5× | 对比测试 | 每月 |
| **商家活跃度** | 日活商家数占注册商家数比 | DAU ÷ 注册数 × 100% | ≥ 40% | tracking_events 表 | 每周 |
| **Agent 利用率** | 各 Agent 实际工作时间占在线时长比 | 工作时长 ÷ 在线时长 | ≥ 60% | agent_tasks 表 | 每日 |
| **月度 API 成本** | LLM API 调用总费用 | Σ 各模型调用费用 | ≤ ¥3,000 | LLM API 计费 | 月度 |

> 与 PRD v1.1 §16.1 验收标准的映射：工作流完成率 → 验收项 #5（自动解决率 ≥ 60%）；平均时长 → 验收项 #3（响应时间 < 10s）

## 第二层：过程质量指标

| 指标名 | 定义 | 告警阈值 | 对应 PRD 章节 | 测量方式 |
|--------|------|:------:|-------------|---------|
| **意图识别准确率** | Manager 正确理解商家意图的比例 | < 90% 告警 | §11.2.1 准确性维度 | 评测集 + 人工抽检 |
| **任务分发准确率** | 任务被正确派发给合适 Agent 的比例 | < 95% 告警 | §7 Agent 配置 | agent_tasks 关联分析 |
| **并行执行成功率** | 并行任务全部成功的比例 | < 85% 告警 | §6.2 Agent 协作模式 | workflow 步骤状态 |
| **首次输出通过率** | Manager 首次质检就通过的比例 | < 70% 告警 | §11.2.1 评分标准 | 质检记录 |
| **自动解决率（客服）** | 无需人工介入的咨询占比 | < 55% 告警 | §16.1 验收 #5 | session 表统计 |
| **内容采纳率** | 运营直接使用或微调后使用的比例 | < 65% 告警 | §16.1 验收 #7 | content_generations 表 |
| **安全合规通过率** | 评测集中安全一票否决的比例 | > 0.1% 告警 | §11.2.1 一票否决 | 评测记录 |
| **异常升级率** | 需要人工介入的异常比例 | > 10% 告警 | §18 风险矩阵 | workflow 异常记录 |

## 第三层：技术健康指标

| 指标名 | 定义 | 告警阈值 | 对应 PRD 章节 | 监控方式 |
|--------|------|:------:|-------------|---------|
| **MCP 工具调用成功率** | MCP 工具调用成功比例 | < 99% | §6.3 MCP 协议集成 | mcp_tool_call 事件 |
| **平均 Token 消耗/工作流** | 每次工作流消耗的 Token 数 | > 预算 120% | §7.3 成本预估 | LLM API 响应 |
| **Agent 响应延迟 (P95)** | 子 Agent 的平均响应时间 | > 10s | §5 性能要求 | agent_tasks 表 |
| **幻觉率** | 生成事实性错误内容的比例 | > 3% | §12.2 幻觉治理 | Bad Case 标注统计 |
| **RAG 召回率 (Top-5)** | 知识库检索前 5 条命中率 | < 90% | §11.2 评测维度 | 评测集 |
| **主备模型 Failover 次数** | 主模型不可用自动切换次数 | > 3 次/周 | §7.3 选型结论 | failover 日志 |
| **向量数据库延迟 (P99)** | Milvus/Chroma 查询延迟 | > 500ms | §5 非功能需求 | RAG 调用监控 |

## 指标看板设计

### 实时看板（Dashboard 第三列）

| 区域 | 展示内容 | 更新频率 |
|------|---------|:------:|
| 核心指标卡 | 今日销售额 / 订单量 / 客单价 / 转化率 | 每 5 分钟 |
| 趋势图 | 近 7 天销售额折线 | 每日 |
| Agent 状态 | 7 个 Agent 的在线/工作中/异常 | 实时 |
| 工作流进度 | 当前活跃工作流 + 步骤状态 | 实时 |

### 运营周报（每周一自动生成）

| 模块 | 内容 |
|------|------|
| 核心指标趋势 | GMV / 订单量 / 客单价 周环比 |
| Agent 效率 | 各 Agent 利用率 + 瓶颈分析 |
| 客服质量 | 自动解决率 / 满意度 / 转人工率趋势 |
| 成本分析 | API 费用 / ROI |
| 异常清单 | 本周告警汇总 + 处理状态 |
